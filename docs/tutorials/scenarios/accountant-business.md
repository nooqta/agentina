# Scenario · Accountant ↔ Small business — books without attachments

**Hook.** "Your accountant has your bank login. Read that sentence again. Here's the version where they get a folder of receipts for exactly one month, and you get a bookkeeper you can question at 11pm."

1. **Pair** — one invite link over any chat.
2. **Business side** — contact → *Share something* → **A folder** → the invoices folder → look only → 1 week *(say: "monthly books = one re-share per month; access that expires like milk")*. The countdown is visible from second one; when it hits zero the access is gone without anyone remembering to remove it.
3. **Accountant side** — *My agents* → *+ New agent*: `bookkeeper`, its folder is where client docs arrive, purpose *"answer questions about this client's books"*. Then share it back: contact → *Share something* → **One of my agents** → `bookkeeper` → look only.
4. **The payoff** — the business owner asks the `bookkeeper` chip: *"what did we spend on software subscriptions in June?"* Answer with the source file cited, straight from the accountant's machine. **Bonus beat:** the accountant binds `bookkeeper` to its own Telegram bot (agent edit → Channels → *+ Telegram*) — the client texts it like a contact, and every question still lands in both Activity feeds.
5. **The wow** — the share expires on camera (or *Stop* is tapped): the chip vanishes from the client's Ask screen, Activity prints "A temporary share ended". "Next month is a new share. Access never outlives the engagement."

**Recap.** No bank logins, no attachments, an assistant instead of email ping-pong, and access that expires like milk.
